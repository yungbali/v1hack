### Problem: "I don't see the page navigation"

**Check 1**: Verify pages directory exists
```bash
ls -la pages/
```
Should show:

**Check 2**: Restart Streamlit
```bash

# Start again
streamlit run app.py
```

**Check 3**: Clear browser cache
- Hard refresh: Cmd+Shift+R (Mac) or Ctrl+Shift+R (Windows)
- Or open in incognito/private window

**Check 4**: Check Streamlit version
```bash
streamlit --version
```
Should be 1.28.0 or higher

### Problem: "Navigation is there but pages don't load"

**Solution 1**: Check for Python errors in terminal
Look for error messages when clicking a page

**Solution 2**: Verify imports work
```bash
python test_pages.py
```

**Solution 3**: Check data availability
```bash
ls -la data/cached/
```
Should show `dashboard_data.parquet` or `test_cache.parquet`

## Expected Behavior

### When You Click a Page:
1. URL changes to reflect the page
2. Page content loads (may take 1-2 seconds)
3. Sidebar shows page-specific filters
4. Main content area updates
5. Page is highlighted in navigation

### When You Use Filters:
1. Charts update within 1 second
2. KPIs recalculate
3. Data quality warnings may appear
4. Filter state is page-specific (doesn't carry over)

### When You Switch Pages:
1. Previous page state is preserved
2. New page loads with default filters
3. Data is cached (fast loading)
4. No need to reload data
